A single zip file will be available for download. It will include all result files, reference files, and the original input data file. All result data files will be CSV format.

The zip file will included the base name of the original file along with the date and time.

The individual files and their contents are output in four sub-folders. Each output file is described in the table below.‘yyyymmdd’ represents the date when the associated file has last been updated.

Output file Subfolder Description
Original file name _user_input Original input file.
EGLEtools_IBI_Attributes_yyyymmdd reference Regional attribute table.
EGLEtools_IBI_Attributes_Metadata_yyyymmdd reference Metadata for the regional attribute table.
EGLEtools_IBI_Taxa_Translator_yyyymmdd reference Regional taxa translator table.
EGLEtools_IBI_Taxa_Translator_Metadata_yyyymmdd reference Metadata for the regional taxa translator table.
IBI_TaxaTranslator_modify quality_control Lists the taxa that occur in the input file, plus: 1) original TaxonID; 2) ‘translated’ TaxonID (OTU that will be used for the calculation); 3) whether the original TaxonID occurs in the regional taxa translator table (TRUE/FALSE); 4) whether the original TaxonID was changed during the ‘translation’ process (YES/NO); 5) total count in the input file (# individuals summed across samples).
IBI_TaxaTranslator_nonmatch quality_control Lists taxa in the input file that don’t occur in the regional taxa translator table (if the file is blank, all are ‘matches’).
IBI_TaxaTranslator_source quality_control Provides documentation of the Operational Taxonomic Unit (OTU) and source files that were used.
IBI_StreamClassification quality_control List of samples assigned to IBI Stream Classes. Contains interim fields necessary for index classification.
IBI_FileBuilder_CompleteInput results_EGLE_FileBuilder_Output Final output, with ‘translated’ TaxonID, NonTarget designations, attributes and phylogeny, and assigned Index Classes. The file is ready to be run through the IBI calculator.
